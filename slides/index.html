Sass + Compass

Sass

Sass is a language construct to create CSS

It is an easier way to write awesome CSS

Sass homepage

Syntax


Sass has TWO syntaxes:

.sass and .scss

.sass - whitespace sensitive



	// .sass
	.my-selector
		font-size: 2em
		background: #333
		font-family: $monospace



	// .scss
	.my-selector {
		font-size: 2em;
		background: #333;
		font-family: $monospace;
	}


.scss - superset of CSS


Today we're using SCSS


Code reasons

  1. Easier to understand if you know CSS
  2. Easier to integrate in the real-world


Team reasons

  • Helps onboard CSS-savvy teammates.
  • SCSS allows you to convert a project
    piece by piece even in-progress

Variables



	$blue: #00ff00;
	$monospace: "Source Code Pro", "Courier New", monospace;
	$default-fontsize: 1em;
	$default-padding: 2 * $default-fontsize;

Sass lets you use variables to store values.

Functions



	@function pixel-to-em($pixel, $font-size: 16px) {
	  @return $pixel / $font-size * 1em;
	}

Functions are just like PHP. They return a single value.

we rarely need functions

Mixins


	// Define
	@mixin button-make($color: #66B360) {
	  background-color: $color;
	  border: 2px solid lighten($color, 20%);
	}

	// Call
	.button-one {
	  @include button-make;
	}

	// Call with arguments
	.button-two {
	  @include button-make($color-facebook);
	}

Mixins are similar, except they return blocks of CSS.

we use these often

Extendables


// Sass
.message {
	margin: 1em;
	padding: 1em;
	font-size: 1.2em;
}
.error {
	@extend .message;
	color: #060;
}
						

/* CSS */
.message, .error {
	margin: 1em;
	padding: 1em;
	font-size: 1.2em;
}
.error {
	color: #060;
}


Extendables allow you to build upon a common style.

we use these most often, as they allow
for smaller CSS files than mixins

Comments


	/*

Media Queries

Example media queries



	// If the window is at least 500px wide...
	@media (min-width: 500px) { ... }

	// If the window is less than 785px wide...
	@media (max-width: 785px) { ... }

	// If the user is printing the page...
	@media (print) { ... }

	// If the window is in between 520px and 699px...
	@media all and (min-width: 520px) and (max-width: 699px) { ... }

Breakpoint

Breakpoint is a Compass extension. It makes
media queries much easier to write and read.


// Define a breakpoint
$nav-lg: 682px;

.primary-content {
	width: 100%;

	// Use breakpoint
	@include breakpoint($nav-lg) {
		width: 60%;
		margin-right: 4%;
	}
}

.primary-content {
  width: 100%;
}
@media (min-width: 682px) {
	.primary-content {
		width: 60%;
		margin-right: 4%;
	}
}

No-query fallbacks

Breakpoint can also output fallback classes to use with IE since media queries did not work before IE9


$breakpoint-no-query-fallbacks: true;
$nav-lg: 500px, 'no-query' '.lte-ie9';

nav {
	@include breakpoint($nav-lg) {
		width: 60%;
		margin-right: 4%;
	}
}

						

@media (min-width: 500px) {
	nav {
		width: 60%;
		margin-right: 4%;
	}
}
.lte-ie9 nav {
	width: 60%;
	margin-right: 4%;
}

Singularity & The Grid

The 960 Grid System

Create a layout with 12 columns, be able to divide those columns into halves, thirds, fourths, sixths, etc.


Easy.


We can create a simple layout like this within Singularity.

Basic Variables

  • $grids: All of the grid information, we will start with a basic layout (12 columns).

  • $gutters: The gutter information for the grids we are using. Expressed as a fraction of a single column.

  • $output: How we want to output the grid, 960 relies on 'float'

$grids: 12;
$gutters: 1/4;
$output: 'float';

Creating a Layout

We output a column with: 

@include grid-span($span, $position);

Where $span is the number of columns, and $position is where in the grid to position the object


#container {
  max-width: 60em;
  clear: both;
}

#left {
  @include grid-span(6,1);
}

#right {
  @include grid-span(6,7);
}

Aurora: Theme and Compass Extension

What is Aurora?

Aurora is two things. It is a base theme in Drupal, written to provide only the HTML5 templates you need, some small tools, and nothing more. It let's you be in control of your theme.


It is also a compass extension, designed to allow you to create you own sub-themes without the need to copy a starter-kit.

The Compass Extension

The compass extension provides three flavors of an Aurora sub-theme to choose from. They are all very similar, only their Sass partial structure differs:

  • Aurora Proper: The standard Aurora base-theme.

  • Corona: A variation, containing more variables, and structure to start your development off with.

  • Polaris: A SMACCS-based sub-theme

Creating a Sub-Theme

On the machine you are using, you must have ruby, compass, and compass-aurora all installed. Full directions are on the Aurora documentation.


To create a sub-theme, you can use the following command-line code:

compass create {theme_name} -r aurora --using aurora/corona

Challenge:

Bronze: Create a Corona-based Aurora sub-theme.


Silver: Enable that theme within Drupal settings page.


Gold: Change the output directories within Compass create to use "js" instead of "javascripts".
